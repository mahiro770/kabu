# 株式AI分析ツール 設計書

最終更新: 2026-07-09

## 1. システム概要

日本株・米国株を対象としたAI支援の株式分析Webアプリケーション。Streamlitのマルチページ機能を用いて構築されており、ローソク足チャート・多数のテクニカル指標・財務指標・AIによる自然文の分析レポート・ニュース駆動の銘柄スクリーニング・複数階層のウォッチリスト機能を1つのアプリとして提供する。

- **公開URL**: https://kabusearch.streamlit.app/
- **リポジトリ**: https://github.com/mahiro770/kabu
- **想定利用者**: 個人投資家（開発者本人およびURLを共有した知人・コミュニティ）
- **表示言語**: 日本語デフォルト、一部ページは英語表示にも対応（`app.py`のみ。`pages/`配下は日本語固定）

### 1.1 提供価値

| 課題 | 提供する解決策 |
|---|---|
| 個別銘柄のテクニカル・財務状況を素早く俯瞰したい | 1画面でローソク足・8種のシグナル・財務指標をまとめて表示 |
| 大量の指標を自分で解釈するのが大変 | ローカルLLM/Gemini APIによる自然文レポート生成 |
| 日々のニュースから注目銘柄を探すのが大変 | ニュース見出しをAIに要約・ランキングさせるスクリーニング機能 |
| 気になる銘柄を仲間と共有したい | 合言葉制の個人/コミュニティ/公開の3階層ウォッチリスト |

### 2.1 実行モデル

Streamlitはファイルベースのマルチページアプリで、ユーザー操作のたびに該当ページのPythonスクリプトを**最初から最後まで再実行**する（リアクティブ実行モデル）。ページ間で状態を引き継げるのは `st.session_state` のみであり、DBやファイルには依存しない。本アプリでは以下の状態を `st.session_state` に保持する。

| キー | 内容 |
|---|---|
| `watchlist` | 公開ウォッチリストの中身（起動時にFirestoreからロード） |
| `current_ticker` | 分析対象として選択中のティッカー（ウォッチリストページの銘柄名クリックで設定→`app.py`に`st.switch_page`で遷移） |
| `personal_watchlist_owner` / `personal_watchlist` | 個人リストの解錠済みユーザー名とその中身 |
| `community_watchlist_owner` / `community_watchlist` | コミュニティリストの解錠済みコミュニティ名とその中身 |

### 2.2 デプロイ構成

- ホスティング: Streamlit Community Cloud（GitHubリポジトリと連携し、`main`ブランチへのpushで自動再デプロイ）
- シークレット管理: Streamlit CloudのSecrets機能（TOML形式）に `GEMINI_API_KEY` と `[firestore_credentials]`（GCPサービスアカウントJSONの中身）を設定
- ローカル開発: `.env`ファイル + `GOOGLE_APPLICATION_CREDENTIALS`（サービスアカウントJSONファイルパス）
- 認証情報の解決順序（`src/watchlist.py`）: ① `GOOGLE_APPLICATION_CREDENTIALS`のファイルパス → ② Streamlit Secretsの`firestore_credentials`テーブル → ③ デフォルトのADC（Application Default Credentials）

## 3. 画面構成・機能一覧

### 3.1 `app.py`（メインページ：銘柄分析）

**サイドバー**
- ユーザー名入力
- ウォッチリストタブ: 🌐公開 / 🏘️コミュニティ / 👤個人（コミュニティ・個人は合言葉制、忘れた場合は名前指定で誰でも再設定可能）
- 設定: AIモデル選択（Ollama検出時はOllamaモデル一覧＋Gemini利用可能なら追加、両方候補になり得る）、表示言語（日本語/English）、分析期間（3ヶ月〜5年）

**メインエリア**
1. 銘柄コード/会社名の入力欄＋「分析」ボタン（数字4桁は日本株`.T`として、それ以外は米国株として解決。ティッカー直接ヒットしなければ会社名検索にフォールバック）
2. ヘッダー指標: 現在値・52週高値/安値・RSI(14)・ボラティリティ・総合シグナルバッジ
3. タブ構成
   - **📈価格チャート**: ローソク足＋オーバーレイ選択（MA20/50/200・ボリンジャーバンド・一目均衡表・VWAP）、8指標のシグナルバッジ一覧、日経平均/S&P500との比較チャート＋期間別超過リターン
   - **📊インジケーター**: RSI・MACD・ADX・OBVの各チャート、最新テクニカル値の一覧表
   - **💹財務指標**: バリュエーション（PER/PBR/EPS/時価総額）、収益性（ROE/ROA/各種利益率）、成長性・財務健全性、信用取引（日本株のみ、株探スクレイピング）、過去の業績推移、アナリスト業績予想・目標株価、会社概要、役員一覧、大株主一覧、関連リンク
   - **🤖AI分析**: ボタン押下でAIモデルがテクニカル・財務・シグナルを総合したレポートをストリーミング生成
4. 未分析時: ウォッチリスト一覧ページへのリンク＋使い方ガイド

### 3.2 `pages/1_注目銘柄スクリーニング.py`

- サイドバー: AIモデル複数選択（比較用）、応答タイムアウト（30〜300秒）、日経/Yahoo!の見出し取得件数スライダー
- 実行ボタンで日経マーケット速報＋Yahoo!ニュース経済RSSの見出しを取得し、選択した各AIモデルに並行して「翌営業日以降に注目すべき銘柄」をランキング形式で生成させる（モデルごとにタブ分け、応答が遅い/止まったモデルはタイムアウトで打ち切り、他のモデルの結果表示をブロックしない）

### 3.3 `pages/2_ウォッチリスト一覧.py`

- 共通の期間切り替え（1日/1週/1カ月/6カ月/1年/2年、1日・1週はイントラデイ足）
- 🌐公開 / 🏘️コミュニティ / 👤個人 の3タブ（コミュニティ・個人はホーム画面で解錠済みの場合のみ表示）
- 各銘柄をカード表示: 銘柄名（クリックで`app.py`に遷移し分析実行）、現在値・値動き、価格チャート（軸・日付ラベル付き）、メモ欄（数行）、フォルダ（自由入力のグループ名、保存すると同名でグルーピング）
- フォルダは選択式（ラジオボタンでフォルダを切り替え、フォルダ未設定の銘柄は「その他」に集約）

| `data_fetcher.py` | yfinanceラッパー。`get_stock_data`（OHLCV、`interval`指定可）、`get_stock_info`、`get_financial_history`、`get_earnings_forecast`、`get_major_holders`。失敗時は例外を投げず`None`/`{}`/`[]`を返す |

| `technical_analysis.py` | 純粋関数群。`add_indicators`でMA/BB/RSI/MACD/一目均衡表/ADX/OBV/VWAPを算出、`get_signals`で指標ごとの買い/売り/中立判定と総合シグナル、`get_summary_stats`で52週高安値・期間リターン・出来高・ボラティリティを算出 |

| `stock_screener.py` | スクリーニング用プロンプト構築とストリーミング。`screen_stocks_stream_with_timeout`はスレッド+キューで応答タイムアウトを実現 |

### 4.1 テクニカル指標・シグナルロジックの詳細

`add_indicators`が算出する指標:

| 指標 | 内容 |
|---|---|
| MA20/50/200 | 単純移動平均 |
| ボリンジャーバンド | 20日MA ± 2σ |
| RSI(14) | Wilder方式のEWMベース |
| MACD | EMA12−EMA26、シグナルはEMA9、ヒストグラムはその差 |
| 一目均衡表 | 転換線(9)・基準線(26)・先行スパンA/B（26日先シフト、チャート描画時はさらに26日先まで雲を投影）・遅行スパン（26日前シフト） |
| ADX/+DI/-DI | Wilder方式のEWM平滑化（period=14） |
| OBV | 出来高バランス＋20日EMA |
| VWAP | 20日ローリング出来高加重平均価格 |

`get_signals`の総合判定（`DIRECTIONAL_SIGNAL_KEYS`＝8指標: ma_cross, rsi, macd, bb, ichimoku, adx, obv, vwap）:

- 買い/売りが8指標中75%以上（6/8以上）→「買い」/「売り」
- 過半数が買い側/売り側 →「やや買い」/「やや売り」
- それ以外 →「中立」

## 5. データ設計（Firestore）

Firestoreネイティブモード（Standardエディション、`asia-northeast1`）を使用。

| コレクション | ドキュメントID | スキーマ |
|---|---|---|
| `public_watchlist` | `default`固定 | `{"items": [...]}` |
| `personal_watchlists` | ユーザー名 | `{"passphrase_hash": <sha256 hex \| null>, "items": [...]}` |
| `community_watchlists` | コミュニティ名 | 同上 |

**`items`配列の各要素（銘柄エントリ）:**

```json
{
  "ticker": "7203.T",
  "name": "トヨタ自動車株式会社",
  "memo": "決算待ち",       // 任意
  "group": "自動車産業",     // 任意（未設定時はUI上「その他」に分類）
  "added_by": "山田"        // 公開・コミュニティリストのみ
}
```

- 合言葉は平文保存せず SHA-256 ハッシュ化して保存
- 未登録の名前に対する`_verify_passphrase`は常に`True`を返す（＝「初回登録＝好きな合言葉で作成可」という設計）
- 合言葉紛失時のリセットは本人確認を行わず、名前さえ分かれば誰でも再設定可能（低リスクな用途を前提とした割り切り。既存の`items`は保持される）

## 6. 外部連携仕様

| 連携先 | 用途 | 認証方式 | 失敗時の挙動 |
|---|---|---|---|
| Yahoo Finance（yfinance） | 株価・財務データ取得 | 不要 | `None`/`{}`を返し、UI側でエラー表示 |
| 株探（kabutan.jp） | 信用取引残高・大株主（日本株のみ） | 不要（HTMLスクレイピング） | `None`/`[]`を返す |
| 日本経済新聞・Yahoo!ニュース | ニューススクリーニング用見出し | 不要（スクレイピング/RSS） | `[]`を返す |
| Ollama | ローカルLLMによるAI分析 | 不要（ローカルサーバー） | 接続不可なら選択肢から除外／エラーメッセージをインライン表示 |
| Google Gemini API | クラウドLLMによるAI分析（Ollama不要環境向け） | `GEMINI_API_KEY` | 同上 |
| Google Cloud Firestore | ウォッチリスト永続化 | サービスアカウントJSON（環境変数 or Streamlit Secrets） | 例外はそのまま送出（現状リトライ/フォールバックなし） |
| deep-translator（Google翻訳） | 会社名・事業概要の日本語訳 | 不要 | 例外時は原文をそのまま表示（`@st.cache_data`でキャッシュ） |

## 7. 非機能要件・設計方針

- **エラーハンドリング方針**: 外部I/O（yfinance・スクレイピング・翻訳・AI）は例外を握りつぶし`None`/`[]`/元の値へフォールバックする。呼び出し側は常に空値を考慮した表示分岐を行う。
- **キャッシュ戦略**: `st.cache_data`を用途別TTLで使い分け（会社名翻訳24時間、事業概要翻訳1時間、ウォッチリストの株価取得5分）。過度なAPI呼び出し・スクレイピング頻度を抑制する。
- **セキュリティ**:
  - Firestoreサービスアカウントキーは`.gitignore`済みでリポジトリに含めない
  - 合言葉はハッシュ化して保存（平文非保持）
  - ただしパスフレーズ制は「なりすまし防止」レベルの軽量な仕組みであり、強固な認証基盤ではない（想定利用規模に対する意図的なトレードオフ）
- **国際化**: `app.py`のみ日本語/英語のインラインテキスト切り替え（i18nフレームワーク不使用）。`pages/`配下は日本語固定
- **パフォーマンス**: AIストリーミング応答は各チャンクを逐次UIに反映。スクリーニングは複数モデルをタイムアウト付きスレッドで実行し、遅いモデルが全体をブロックしないようにしている

## 8. 制約・既知の課題

- Firestoreへの書き込み失敗時のリトライ機構がない
- 合言葉紛失時の再設定に本人確認要素がなく、意図的に「誰でも再設定可能」としている（コミュニティ利用の性質上の割り切り）
- 一目均衡表の雲描画のみ、指標データフレーム自体を未来方向に拡張する例外的な実装になっている（チャート描画専用、シグナル計算には影響しない）
- Streamlitの内部DOM実装（テストID等）はバージョン間で変化するため、CSSカスタマイズの一部は複数バージョン互換のセレクタを併記している
- Ollama/Gemini双方が利用不可の場合、AIモデル選択肢が空になりAI関連機能が使えない
